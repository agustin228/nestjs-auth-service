# Auth Service

`Auth Service` is a microservice will handling `Auth of Users` to enable the user interact to other service.

## Prerequisites

1. Node.js >= `18.16.0`
2. Typescript >= `5.1.3`
2. PostgreSQL >= `14`
3. NPM >= `9.5.1` / yarn >= `1.22.19` / pnpm >= `8.6.2`
4. Git >= `2.34.1`
5. Docker >= `24.0.2` + Docker Compose >= `2.19.1`
6. Text Editor/IDE (VSCode/Webstorm)
8. Postman

## Project Structure
Project structure follows the `nestjs` pattern that generated by `nestjs/cli`. 

Notes: This pattern using `monorepo` that merge `config` and `auth service` into one project. Config place to `libs` folder and auth service place to `src` folder in parent directory. 

```
├── README.md
├── libs
│   └── common
│       ├── src
│       │   ├── config
│       │   │   ├── config.module.ts
│       │   │   └── config.schema.ts
│       │   ├── database
│       │   │   └── database.module.ts
│       │   └── index.ts
│       └── tsconfig.lib.json
├── nest-cli.json
├── node_modules
├── package.json
├── pnpm-lock.yaml
├── src
│   ├── auth
│   │   ├── auth.controller.spec.ts
│   │   ├── auth.controller.ts
│   │   ├── auth.module.ts
│   │   ├── auth.service.spec.ts
│   │   └── auth.service.ts
│   └── main.ts
├── test
│   ├── app.e2e-spec.ts
│   └── jest-e2e.json
├── tsconfig.build.json
└── tsconfig.json
```

## Installation

```bash
# Using pnpm
$ pnpm install

# Using yarn
$ yarn install

# Using npm
$ npm install
```

## Running the app

```bash
# development
$ pnpm start # or
$ yarn start #or
$ npm run start

# watch mode
$ pnpm start:dev # or
$ yarn start:dev # or
$ npm run start:dev

# production mode
$ pnpm start:prod # or
$ yarn start:pro # or
$ npm run start:dev
```
